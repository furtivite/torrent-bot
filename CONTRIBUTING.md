## Contribution Guide

Спасибо за интерес к этому проекту.

Этот бот используется в реальном домашнем окружении, поэтому приём изменений максимально консервативный: **приоритет — стабильность, безопасность и простота.**

---

## Базовые правила

- Перед любыми крупными изменениями сначала заводите issue и обсудите идею.
- Не ломайте существующее поведение без очень веской причины.
- Любые изменения, влияющие на безопасность (обработка `.torrent`, работа с диском, авторизация), должны быть:
  - минимальными и хорошо мотивированными;
  - прикрыты тестами.
- Не добавляйте тяжёлые зависимости и фреймворки.
- Поддерживайте текущую структуру проекта: `opt/torrent-bot/`, `handlers/`, `utils/`, `docs/`.

---

## Как поднять проект для разработки

1. Клонировать репозиторий.
2. Создать виртуальное окружение (Python 3.12) и установить зависимости:

   ```bash
   python3.12 -m venv .venv
   source .venv/bin/activate
   pip install --upgrade pip
   pip install -r requirements.txt
   ```

3. Для запуска тестов реальные секреты не нужны — используются заглушки (см. GitHub Actions workflow).  
   Локально можно запускать:

   ```bash
   pytest
   ```

4. Для ручного запуска бота — следуйте инструкциям в `README.md` и `docs/setup-ubuntu-ssh.md`.

## Стиль кода

- Python 3.12, без сторонних стайлгайд‑фреймворков.
- Маленькие функции и модули с чёткой ответственностью.
- Предпочитайте явную обработку ошибок, без «голых» `except Exception:` без логирования.
- Сообщения пользователю — короткие и по возможности на русском.

---

## Как отправить Pull Request

1. Форкните репозиторий и создайте ветку от `main`/`master`.
2. Внесите изменения с небольшими, осмысленными коммитами.
3. Добавьте/обновите тесты, если меняете поведение.
4. Убедитесь, что локально проходят:

   ```bash
   pytest
   ```

5. Обновите документацию при необходимости:
   - `README.md` — если меняется поведение/интерфейс;
   - файлы в `docs/` — если меняются сценарии установки/архитектура.
6. Создайте PR, кратко опишите:
   - **что** изменено;
   - **почему** это нужно;
   - как протестировано.

---

## Что пока нежелательно менять

- Логику авторизации (`utils/auth.py`) — только по прямой необходимости.
- Жёсткие проверки безопасности в `handlers/documents.py`, `utils/torrent_layout.py` и т.п.
- Формат сообщений пользователю, если нет веской UX‑причины.

Если сомневаетесь — лучше сначала открыть issue и обсудить.+
