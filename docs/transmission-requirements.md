## Требования к настройке Transmission

Этот бот управляет **уже настроенным** Transmission BitTorrent‑сервером.  
Ниже — минимальный список требований к конфигурации Transmission.

---

## 1. Доступ по RPC

- Включён RPC‑интерфейс:
  - `rpc-enabled`: `true`;
  - `rpc-bind-address`: обычно `0.0.0.0` или `127.0.0.1`, если бот на том же хосте;
  - `rpc-port`: совпадает с `TRANSMISSION_PORT` в `/etc/torrent-bot.env`;
  - `rpc-username` / `rpc-password`: совпадают с `TRANSMISSION_USERNAME` / `TRANSMISSION_PASSWORD`.
- Любые ограничения по IP/хостам (`rpc-whitelist` и т.п.) должны позволять доступ боту.

Бот использует эти параметры через модуль `config.py` и `transmission_client.py`.

---

## 2. Watch‑директория для `.torrent`‑файлов

- В Transmission настроена директория автозагрузки `.torrent`‑файлов (watch‑dir), например:
  - `/mnt/data/trackers`.
- Эта директория должна совпадать с `TRACKERS_DIR` в `/etc/torrent-bot.env`.
- Права доступа:
  - процесс бота должен иметь право:
    - создавать и переименовывать файлы в `TRACKERS_DIR`;
    - удалять временные файлы (с префиксом `.uploading_`).

Бот сохраняет проверенные `.torrent`‑файлы именно сюда, после всех проверок.

---

## 3. Каталог загрузок

- Основной каталог загрузок Transmission (download‑dir) должен быть согласован с `DOWNLOADED_DIR`:
  - пример: `/mnt/data/downloaded`;
  - этот путь указывается в `/etc/torrent-bot.env` в `DOWNLOADED_DIR`.
- Бот использует `DOWNLOADED_DIR` для проверки:
  - был ли торрент уже скачан;
  - корректны ли размеры файлов на диске.

---

## 4. Права доступа к файловой системе

- У пользователя, под которым запускается Transmission, и пользователя/сервиса, под которым
  запускается бот:
  - должен быть общий доступ к:
    - `TRACKERS_DIR` (watch‑dir);
    - `DOWNLOADED_DIR` (каталог загрузки);
    - при необходимости другим каталогам в `DATA_ROOT`.
- Рекомендуется:
  - использовать общую группу (например, `media` или `torrent`);
  - выдать нужные права чтения/записи для этой группы.

---

## 5. Web UI

- Transmission Web UI должен быть доступен по URL, который указывается в:
  - `WEB_UI_URL` в `/etc/torrent-bot.env` (например, `https://your-host/transmission/web/`).
- Бот:
  - **не** настраивает Web UI и HTTPS;
  - только отправляет пользователю ссылку и учётные данные;
  - может отправлять корневой сертификат, если указан `CERT_FILE_PATH`.

---

## 6. Отдельный HTTPS / reverse proxy

- Если используется HTTPS и домены:
  - настройка TLS, доменов и reverse‑proxy (Caddy, nginx и т.п.) выполняется отдельно;
  - бот лишь:
    - хранит путь к корневому сертификату (`CERT_FILE_PATH`);
    - выдаёт этот файл пользователю и показывает текстовые инструкции по установке.

---

## 7. Проверка работоспособности

После настройки Transmission и окружения бота:

1. Убедитесь, что команда `transmission-remote` (если используется) видит те же настройки, что и бот.
2. Запустите бота и выполните:
   - команду `/torrents` — убедитесь, что список торрентов читается без ошибок;
   - отправьте test `.torrent` — проверьте, что файл попадает в `TRACKERS_DIR` и подхватывается Transmission;
   - отправьте test magnet‑ссылку — убедитесь, что она добавляется в Transmission.
